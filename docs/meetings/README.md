# Architecture Decision Records (ADRs)

These documents are not meeting notes — they record **why** key decisions were
made, so the reasoning survives long after the decision.

Each record uses a simple structure:

- **Decision** — what we chose
- **Reason** — why
- **Alternatives** — what else was considered
- **Status** — Accepted / Superseded / etc.

Files are numbered sequentially: `0001-foundation.md`, `0002-components.md`, ...
