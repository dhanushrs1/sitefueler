# SiteFueler v0.6.0 — Account Security & 2FA

**Type:** Minor (security hardening; builds on the identity milestone)
**Tag:** `v0.6.0`

Mandatory two-factor authentication for staff, recovery codes, trusted devices,
a login audit trail, throttling, and a role-aware password policy. Customers
remain unchanged (optional 2FA is future work).

## Two-factor authentication (TOTP)

- Standard **TOTP (RFC 6238)** — works with any authenticator app (Google /
  Microsoft Authenticator, Authy, 1Password, Bitwarden). Not vendor-locked.
- **Mandatory** for roles in `config('authentication.two_factor.required_roles')`
  — `admin`, `super-admin`, `editor`, `support`. Customers are not required.
- **Forced setup wizard** on first sign-in: scan QR → verify first code → save
  recovery codes. An account is only marked `two_factor_confirmed_at` after the
  first code verifies, so staff can never reach the admin panel half-configured.
- **Challenge** on every new session: 6-digit code or recovery code. Pending
  login is stashed; authentication only completes once the second factor passes.
- **Google sign-in** funnels staff through the same challenge / enrollment, so
  OAuth cannot bypass 2FA.

## Recovery codes
- 10 single-use, **hashed** codes shown once at setup.
- Regenerable from the admin **Security** page; old codes are invalidated.

## Trusted devices
- "Remember this device for 30 days" → signed token in an http-only cookie;
  only the SHA-256 hash is stored in `trusted_devices`.
- Trusted devices skip the challenge until they expire.
- Listed and individually (or fully) revocable from the Security page.

## Login history
- Append-only audit trail (`login_history`) of every attempt — success and
  failure — with IP, browser, and device. Shown on the Security page.
- Dependency-free UA parsing helper (`App\Support\DeviceInfo`).

## Hardening
- **Login throttling** per email + IP — customers `5/15 min`, admins `5/30 min`.
  2FA challenge is independently limited to `5/15 min`.
- **Role-aware password policy** (`App\Support\PasswordRules`):
  - Customer: min 8 (the `Password::defaults()` baseline)
  - Admin: min 12, mixed case, number, symbol, plus an `uncompromised()` check
- **Admin idle timeout** (`ADMIN_SESSION_LIFETIME`, default 30 min) on top of
  the global session lifetime.
- Session ID + CSRF token regenerated on every successful login.

## Admin middleware stack
```
auth → role:<admin roles> → 2fa.enrolled → 2fa.verified → admin.timeout
```
Each middleware has a single responsibility.

## Schema additions
- `users`: `two_factor_secret` (encrypted), `two_factor_enabled`, `two_factor_confirmed_at`
- `two_factor_recoveries(id, user_id, code_hash, used_at, timestamps)`
- `trusted_devices(id, user_id, token_hash, name, browser, platform, ip, last_used_at, expires_at, timestamps)`
- `login_history(id, user_id?, email, ip, browser, device, location?, successful, created_at)`

## New / changed code
- Services: `TwoFactorService`, `TrustedDeviceService`
- Helpers: `App\Support\{PasswordRules, DeviceInfo, LoginAudit}`
- Middleware: `EnsureTwoFactorEnrolled`, `EnsureTwoFactorVerified`, `EnforceAdminSessionTimeout`
- Controllers: `Auth\TwoFactorSetupController`, `Auth\TwoFactorChallengeController`,
  `Admin\SecurityController`
- Views: `auth/two-factor/{setup,challenge,recovery-codes}`, `admin/security/index`
- CSS: `components/two-factor.css` (tokens only, no hardcoded values)
- Dependencies: `pragmarx/google2fa ^9.0`, `bacon/bacon-qr-code ^3.1`

## Fixed
- Login threw **`RuntimeException: This password does not use the Bcrypt algorithm`**
  when the seeded admin row carried a stale `$2a$` bcrypt hash that this PHP
  build's `password_get_info()` reports as `unknown`. Re-seeding regenerates
  with `$2y$` so Laravel's `BcryptHasher::check()` accepts it.

## Setup
```bash
composer install
php artisan migrate
php artisan db:seed --class=AdminUserSeeder
# add ADMIN_SESSION_LIFETIME=30 and TWO_FACTOR_ISSUER (optional) to .env
```

On first sign-in the seeded admin (`admin@sitefueler.test` / `password`) is
forced through 2FA setup — have an authenticator app ready.
