# SiteFueler v0.1.0 — Foundation Release

**Milestone:** 1 — Foundation
**Status:** Closed ✅
**Tag:** `v0.1.0`

The foundation release. This establishes the architecture, design system,
documentation, and workflow that everything else will be built on. Milestone 1 is
closed; these foundations are stable.

## Highlights

- **Architecture** — Laravel 13 / PHP 8.4 / MySQL, Blade + vanilla CSS + vanilla
  JS. No Tailwind/Bootstrap/jQuery/React/Vue/Alpine/Sass. Static assets served
  from `public/assets/` (no build step) for simple Apache / shared-hosting deploys.
- **Design System** — `docs/design-system.md`: brand identity, color system
  (incl. semantic colors + soft tints), typography, spacing, radius, shadows,
  grid, breakpoints, and design philosophy.
- **CSS Foundation** — token-first architecture:
  `variables.css` → `layout.css` → `utilities.css` → `components.css`
  (frozen `variables.css` as the single source of truth).
- **Component Specifications** — nine specs in `docs/components/` (header, button,
  form, badge, alert, card, table, modal, footer), each with Purpose, Structure,
  Variants, Sizes, States, Behavior, Accessibility, Responsive, Usage Rules, and
  Future Enhancements.
- **Architecture Decision Records** — `docs/meetings/` (0001 Foundation,
  0002 Components, 0003 Principles).
- **Git Workflow** — `main` + short-lived `feature/*`; Conventional Commits;
  Definition of Done; reusability gate; "Prefer stability over speed" principle.
- **Project Docs** — `README`, `CHANGELOG`, `ROADMAP` (milestone-based),
  `CONTRIBUTING`, proprietary `LICENSE`, and GitHub issue/PR templates.

## Documentation map

```
docs/
  design-system.md   Global visual rules and tokens
  components/        Per-component specifications
  architecture/      How the system works
  meetings/          Architecture Decision Records (ADRs)
  reviews/           Component review records
  releases/          Release notes (this folder)
```

## Not included (by design)

- No component implementation beyond the initial Button (reviewed in Milestone 2).
- No pages, authentication, store, or admin panel yet.
