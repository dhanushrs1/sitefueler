# SiteFueler v0.2.1 — UI Refinements

**Type:** Patch (visual refinement of Milestone 2 components)
**Tag:** `v0.2.1`

A polish pass on the UI System for a cleaner, more minimal, more professional
look. No new components; token-level changes that propagate everywhere.

## Changes

### Corner radius (less rounded)
Corners read as too pill-like. Reduced to subtle, professional rounding:

| Token         | Before | After |
| ------------- | ------ | ----- |
| `--radius-sm` | 12px   | 6px   |
| `--radius-md` | 16px   | 10px  |
| `--radius-lg` | 24px   | 14px  |

Applies automatically to buttons, inputs, alerts, cards, tables, and modals.
Badges/toggles keep `--radius-full` (pill).

### Buttons (tighter, minimal)
Default buttons were too large with heavy padding. New compact scale:

| Size   | Height (before → after) | Padding (before → after) | Font            |
| ------ | ----------------------- | ------------------------ | --------------- |
| Small  | 40 → **32px**           | 16 → **12px**            | `--text-small`  |
| Medium | 48 → **40px**           | 24 → **16px**            | `--text-small`  |
| Large  | 56 → **48px**           | 32 → **20px**            | `--text-body`   |

Medium (default) now uses 14px text; only Large uses 16px. All button dimensions
are tokenized (spacing scale + `--button-height`).

## Docs kept in sync
- `design-system.md` §5 (radius), `button.md` §4 (sizes)
- Radius rows in `form.md`, `alert.md`, `card.md`, `modal.md`

## Notes
- `variables.css` was unfrozen only for this owner-requested refinement, then
  re-frozen. No new tokens were added.
- Form control height stays 48px (standard, accessible touch target); only its
  corners are less rounded.
