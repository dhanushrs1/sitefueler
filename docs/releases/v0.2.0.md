# SiteFueler v0.2.0 — Core UI Components

**Milestone:** 2 — Core UI Components
**Status:** Closed ✅
**Tag:** `v0.2.0`

The UI System milestone. Seven reusable, token-based, accessible components that
will power every page of the platform. Each was built through the full lifecycle:
Specification → Implementation → Review → Merge.

## Components

| Component | Highlights |
| --------- | ---------- |
| **Button** | 4 variants, 3 sizes, 6 states, icon support; single parameter-driven Blade component |
| **Form**   | input/textarea/select/checkbox/radio/switch; validation states; label/hint/error; prefix/suffix icons; row/inline layouts |
| **Badge**  | 5 fixed variants, pill shape, optional icon, default labels |
| **Alert**  | success/warning/danger/info; soft tints; dismiss; assertive/polite roles |
| **Card**   | one base card; product/service/blog; media 4:3 + badge overlay; interactive link |
| **Table**  | single admin style; selection/actions/status; hover/selected/empty/loading |
| **Modal**  | native `<dialog>`; sizes; focus trap; Escape; backdrop click; scroll lock |

## Quality bar (applied to every component)

- ✅ Matches its specification document
- ✅ Uses only design tokens (no hardcoded colors/spacing/radii/shadows/type)
- ✅ Responsive (mobile / tablet / desktop)
- ✅ Visible keyboard focus + semantic HTML + appropriate ARIA
- ✅ No duplicated logic across components (Button/Form composed, not re-styled)
- ✅ Reusable Blade markup
- ✅ Reviewed (`docs/reviews/0001`–`0007`)

## Architecture notes

- Component CSS is split under `public/assets/css/components/` and aggregated by
  `components.css` (no monolithic stylesheet).
- A single delegated vanilla-JS file (`app.js`) powers alert dismissal and modal
  control — no framework, no jQuery.
- Blade pattern: one parameter-driven component per family with slots, following
  the Button blueprint.
