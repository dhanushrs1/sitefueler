# SiteFueler v0.3.0 — Layout Components (Application Shell)

**Milestone:** 3 — Layout Components
**Status:** Closed ✅
**Tag:** `v0.3.0`

The application shell — the skeleton every page now sits in: **Header → Main →
Footer**. Built entirely by composing the Milestone 2 components.

## Delivered

| Piece | Notes |
| ----- | ----- |
| **App shell / layouts** | `app` (base) + `marketing` (implemented); `auth`/`dashboard`/`error` placeholders |
| **Header** | sticky 80px; logo, config-driven nav, search, Login (Ghost), Get Started (Primary); mobile hamburger panel |
| **Navigation** | driven by `config/navigation.php`; active-route highlighting; `children` reserved for dropdowns |
| **Search Bar** | reusable, built on the Form System (`type="search"`, `role="search"`) |
| **Footer** | dark, three columns, config links + legal bar; composes Button |
| **Breadcrumb** | ordered list with separators + `aria-current` |
| **Page Title** | heading + subtitle + breadcrumb slot + actions slot |
| **Empty State** | icon + title + message + actions (used by 404) |
| **404** | `errors/404.blade.php` on the `error` layout + Empty State |

## Architecture

- **Composition rule** applied: Header/Footer reuse Button + Form rather than
  re-styling. Nothing reinvents typography or controls.
- **Config-driven navigation** (`config/navigation.php`) — add/remove menu items
  without touching Blade.
- **Page-layout templates** split now to avoid a future refactor.
- New layout-component CSS split under `public/assets/css/components/` and
  aggregated by `components.css`.
