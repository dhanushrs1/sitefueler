# SiteFueler v0.5.1 — Auth Polish & Google Sign-In

**Type:** Minor (completes the identity milestone)
**Tag:** `v0.5.1`

Builds on v0.5.0: working Google Sign-In, a polished auth UI, and a forgot-password
page.

## Google Sign-In
- Socialite + Google credentials; scopes limited to `openid`, `profile`, `email`.
- `SocialAuthService` resolves identity: existing provider → log in; existing email
  → **link** (one account, no duplicates); new → create **customer**.
- Avatar priority: custom → Google → default (never overwrites a custom upload).
- Password nullable for Google users (set one later to enable both methods).
- Google never creates/promotes an admin. Cancelled/invalid flows fail gracefully.

## Auth UI
- Floating-label fields, premium multicolor Google button, refined typography and
  copy across **login / register / forgot password**.
- Full marketing footer retained; fields/buttons sized to 48px.

## Schema
- `roles.description`, `social_accounts.provider_name`, nullable `users.password`.

## Docs
- `docs/architecture/authentication.md` is the frozen identity source of truth.
- Reviews `0014` (auth UI), `0015` (Google auth).

## Setup
```bash
php artisan migrate:fresh --seed
# Google creds live in .env (GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET / GOOGLE_REDIRECT_URI)
```

## Note
The Google consent round-trip should be exercised in a real browser before relying
on it in production.
