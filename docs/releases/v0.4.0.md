# SiteFueler v0.4.0 — Admin Foundation

**Milestone:** 4 — Admin Foundation
**Status:** Closed ✅
**Tag:** `v0.4.0`

The admin shell. Not the full admin panel — the framework every future module
(Templates, Plugins, Services) will plug into.

## Delivered

| Piece | Notes |
| ----- | ----- |
| **Admin login** | email/password, CSRF, admin-only (`is_admin`) |
| **Admin layout** | sidebar + topbar + content; reuses the design system |
| **Sidebar** | config-driven, active state, off-canvas on mobile |
| **Topbar** | burger, page title, notifications + profile dropdowns |
| **Dashboard** | six stat cards (placeholder values) |
| **Breadcrumb** | default + per-page override |
| **Profile** | dropdown + profile page (update name/email) |
| **Logout** | POST + session invalidation |
| **Route protection** | `auth` + `admin` middleware; guests → `admin.login` |

## Architecture

- **Module isolation:** `app/Http/Controllers/Admin/`, `resources/views/admin/`,
  `public/assets/admin/`.
- **Grouped, named routes** (`admin.`) — `Route::resource(...)` for future modules
  drops straight into the `[auth, admin]` group; views use `route('admin.*')`.
- Config-driven sidebar + dashboard (`config/admin.php`).
- New: `is_admin` migration, `AdminUserSeeder`, `EnsureUserIsAdmin` middleware.

## Default admin (dev only)

- Email: `admin@sitefueler.test`
- Password: `password`
- **Change before any non-local deployment.**

## Setup

```bash
php artisan migrate
php artisan db:seed --class=AdminUserSeeder
```
