# SiteFueler v0.5.0 — Identity & Authentication

**Milestone:** 5 — Identity & Authentication
**Status:** Closed ✅
**Tag:** `v0.5.0`

One identity system for everyone (frontend, customer, admin). Roles decide access;
one session for all.

## Delivered

| Area | Notes |
| ---- | ----- |
| **Unified login** | `/login` on the marketing layout — email/password + Continue with Google |
| **Register** | `/register` — public sign-up, always a customer |
| **Roles** | customer / admin / editor / support / super-admin (replaces `is_admin`) |
| **Authorization** | `role:<slugs>` middleware; configurable `ADMIN_PREFIX` |
| **Google sign-in** | Socialite flow + `social_accounts` (multi-provider ready) |
| **Users** | `uuid`, `role_id`, `username`, `avatar`, `status`, login tracking |
| **Header** | auth-aware (guest vs customer vs admin) |
| **Customer dashboard** | `/dashboard` placeholder |

## Architecture
- One `users` table + `roles` + `social_accounts`; one session.
- Public/Google sign-ups → customer; admins created only by admins.
- `provider_data` JSON keeps raw OAuth payloads (forward-compatible).
- Encrypted OAuth tokens; login time + IP recorded.

See `docs/architecture/authentication.md`.

## Setup
```bash
php artisan migrate:fresh --seed
# add GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET to .env to enable Google
```
Default admin (dev): `admin@sitefueler.test` / `password` — **change before deploy**.
